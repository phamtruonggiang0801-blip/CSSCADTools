1.3 Installation Steps / Cac buoc cai dat

⚠
Important / Luu y: Close AutoCAD completely before installation.
Dong AutoCAD hoan toan truoc khi cai dat.

Option A: Automatic Installation (Recommended) / Cai dat tu dong

  1. Right-click install.bat → select "Run as administrator"
    Click chuot phai vao install.bat → chon "Run as administrator"
  2. The script will automatically copy the plugin bundle to the correct folder.
    Script se tu dong copy plugin bundle vao thu muc dung.
  3. Open AutoCAD 2023. The plugin loads automatically.
    Mo AutoCAD 2023. Plugin se tu dong duoc load.
  4. Type MCG_Show in the AutoCAD command line and press Enter.
    Go lenh MCG_Show trong dong lenh AutoCAD roi nhan Enter.

Option B: Manual Installation / Cai dat thu cong

  1. Open Windows Explorer, navigate to:
    Mo Windows Explorer, di den:
    %PROGRAMDATA%\Autodesk\ApplicationPlugins\
-- Thuong la: C:\ProgramData\Autodesk\ApplicationPlugins\
    ℹ
    ProgramData is a hidden folder. Type the path directly in the Explorer address bar.
    ProgramData la thu muc an. Go duong dan truc tiep vao thanh dia chi.
  2. Copy the entire MCGCadPlugin.bundle folder into this directory.
    Copy toan bo folder MCGCadPlugin.bundle vao thu muc nay.
  3. Open AutoCAD 2023 → type MCG_Show to verify.
    Mo AutoCAD 2023 → go MCG_Show de kiem tra.

1.4 Verify Installation / Kiem tra cai dat

In AutoCAD command line, type:
Trong dong lenh AutoCAD, go:
MCG_Show    -- Opens the MCG Tools palette (5 tabs)
MCG_Hide    -- Hides the palette
MCG_Fitting -- Opens palette and focuses Fitting tab
✔
If the MCG Tools palette appears on the right side of AutoCAD with 5 tabs, installation is successful!
Neu palette MCG Tools hien thi ben phai AutoCAD voi 5 tab, cai dat thanh cong!

1.5 Update Plugin / Cap nhat Plugin

To update, close AutoCAD, replace the .dll file inside Contents/, then reopen AutoCAD.
De cap nhat, dong AutoCAD, thay file .dll trong Contents/, roi mo lai AutoCAD.

1.6 Uninstall / Go bo

Simply delete the MCGCadPlugin.bundle folder from ApplicationPlugins.
Chi can xoa folder MCGCadPlugin.bundle khoi ApplicationPlugins.

2. Overview / Tong quan Fitting Management

The Fitting Management tab provides a complete workflow for managing ship fittings — from extracting Inventor geometry to exporting BOM (Bill of Materials) to Excel.
Tab Fitting Management cung cap quy trinh hoan chinh de quan ly fitting tau — tu trich xuat hinh hoc Inventor den xuat BOM ra Excel.
[ Screenshot: Fitting Management tab in MCG Tools palette ]

Workflow Overview / Quy trinh tong the

STEP 1 3D Extraction — Import .idw files

Extract geometry and properties from Inventor IDW drawing files. Outputs: DWG + JSON.
Trich xuat hinh hoc va thuoc tinh tu file ban ve Inventor IDW. Dau ra: DWG + JSON.

STEP 2 Block Fitting — Import .json files

Import the extracted JSON/DWG, create AutoCAD blocks with attributes, and register to Master Catalog.
Import JSON/DWG da trich xuat, tao block AutoCAD voi attributes, dang ky vao Master Catalog.

STEP 3 Fitting Library

Search, browse, and insert standard fittings from the Central Library into drawings.
Tim kiem, duyet va chen fitting tieu chuan tu Thu vien Trung tam vao ban ve.

STEP 4 BOM Export & Ballooning

Scan drawings, count fittings, assign positions, auto-balloon, and export BOM matrix to Excel.
Quet ban ve, dem fitting, gan vi tri, tu dong balloon, xuat BOM matrix ra Excel.

TOOLS Block Utilities

Rename, replace, redefine, add/extract entities from blocks without breaking references.
Doi ten, thay the, cap nhat, them/tach entity tu block ma khong lam hong tham chieu.

Library Folder / Thu muc thu vien

All fitting data is stored at:
Toan bo du lieu fitting duoc luu tai:
C:\Temp_BIM_Library\
    MasterCatalog.json    -- Catalog chinh chua tat ca fitting
    CAS-1234567.dwg      -- File DWG cua tung fitting
    CAS-1234567.json     -- Metadata JSON tuong ung

3. Step 1: 3D Extraction — Import .idw files

⚠
Requirement: Autodesk Inventor must be installed on this machine.
Yeu cau: May tinh phai cai dat Autodesk Inventor.

What it does / Tinh nang lam gi

This feature connects to Inventor via COM automation, opens each .idw file, extracts drawing views and iProperties (Part Number, Description, Mass, Material, Revision, Designer, Title), exports the drawing to DWG format, and saves metadata as JSON.
Tinh nang nay ket noi Inventor qua COM automation, mo tung file .idw, trich xuat drawing views va iProperties (Part Number, Description, Mass, Material, Revision, Designer, Title), export ban ve sang DWG, va luu metadata thanh JSON.

How to use / Cach su dung

  1. In AutoCAD, open the MCG Tools palette → go to the Fitting Management tab.
    Trong AutoCAD, mo palette MCG Tools → vao tab Fitting Management.
  2. Click "Import .idw files" in the 3D EXTRACTION (STEP 1) section.
    Click "Import .idw files" trong phan 3D EXTRACTION (STEP 1).
  3. In the file dialog, select one or more .idw files. You can select multiple files at once.
    Trong hop thoai, chon mot hoac nhieu file .idw. Co the chon nhieu file cung luc.
  4. Wait for Inventor to process each file. A progress message will appear when done.
    Doi Inventor xu ly tung file. Thong bao ket qua se hien len khi xong.

Output / Ket qua

For each .idw file, two files are created in C:\Temp_BIM_Library\:
Voi moi file .idw, 2 file duoc tao trong C:\Temp_BIM_Library\:
FileDescription / Mo ta
{PartNumber}.dwgGeometry exported from Inventor / Hinh hoc export tu Inventor
{PartNumber}.jsonMetadata (iProperties + view dimensions) / Metadata (iProperties + kich thuoc view)
ℹ
If Inventor is already open, the plugin will use the running instance. Otherwise, it will start Inventor in the background and close it when done.
Neu Inventor da mo, plugin se dung instance dang chay. Nguoc lai, se khoi dong Inventor ngam va dong khi xong.

4. Step 2: Block Fitting — Import .json files

What it does / Tinh nang lam gi

Reads the JSON metadata files (from Step 1) along with their corresponding DWG files. Creates block definitions in the current AutoCAD drawing, injects standard attributes, assigns the correct layer, and automatically registers the fitting to the Master Catalog.
Doc file JSON metadata (tu Step 1) cung file DWG tuong ung. Tao block definitions trong ban ve AutoCAD hien tai, chen cac attributes chuan, gan layer dung, va tu dong dang ky fitting vao Master Catalog.

How to use / Cach su dung

  1. In the BLOCK FITTING (STEP 2) section, select the Target BOM Type:
    Trong phan BLOCK FITTING (STEP 2), chon Target BOM Type:
    OptionWhen to use / Khi nao dungLayerColor
    Panel (Structure)Fittings for structural panels / Fitting cho panel ket cauMCG_Fitting_PanelBlue (5)
    Detail (Hull)Fittings for hull details / Fitting cho chi tiet vo tauMCG_Fitting_DetailRed (1)
  2. Click "Import .json files".
    Click "Import .json files".
  3. Select one or more .json files. Each JSON must have a .dwg file with the same name in the same folder.
    Chon mot hoac nhieu file .json. Moi JSON phai co file .dwg cung ten trong cung folder.
  4. The plugin creates blocks and shows a summary of results.
    Plugin tao block va hien thi tom tat ket qua.

Attributes Injected / Cac Attribute duoc chen

Each block created will have these standard attributes:
Moi block duoc tao se co cac attribute chuan sau:
Attribute TagSourceVisible
PART_IDPartNumber from JSONYes
DESCDescription from JSONYes
MASSMass from JSONNo (hidden)
MATERIALMaterial from JSONNo (hidden)
REVISIONRevision from JSONNo (hidden)
BOM_TYPESelected radio optionNo (hidden)
POS_NUMEmpty (assigned later)No (hidden)
⚠
If a block with the same name already exists in the drawing, it will be skipped (not overwritten).
Neu block cung ten da ton tai trong ban ve, no se bi bo qua (khong ghi de).

5. Step 3: Fitting Library

What it does / Tinh nang lam gi

A full-featured catalog manager for browsing, searching, and inserting fittings from the Central Library.
Trinh quan ly catalog day du de duyet, tim kiem, va chen fitting tu Thu vien Trung tam.

How to open / Cach mo

Click "Open Fitting Library" in the Fitting Management tab.
Click "Open Fitting Library" trong tab Fitting Management.

Library Modes / Cac che do

ModeDescription / Mo ta
Master Library The central fitting catalog shared by all projects. You can add, remove, and manage accessories here.
Catalog fitting trung tam dung chung cho tat ca du an. Co the them, xoa, quan ly phu kien o day.
Project Library A project-specific subset. You can edit Position numbers and auto-assign positions here.
Tap con rieng cho tung du an. Co the sua so Position va tu dong gan position o day.

Key Features / Tinh nang chinh

ButtonFunction / Chuc nang
Add from CADSelect geometry or block in AutoCAD and add as a new virtual item to the library.
Chon hinh hoc hoac block trong AutoCAD va them vao thu vien nhu item ao.
Sub-BOM / AccessoriesManage child items (accessories) attached to a parent fitting.
Quan ly item con (phu kien) dinh kem vao fitting cha.
Insert to CADInsert the selected fitting into the current drawing at a pick point.
Chen fitting da chon vao ban ve hien tai tai diem chon.
Add to ProjectCopy selected items from Master to the active Project Library.
Copy item da chon tu Master sang Project Library dang hoat dong.
Auto-Assign PosAutomatically assign position numbers (001, 002, ...) to Detail fittings in Project mode.
Tu dong gan so position (001, 002, ...) cho fitting Detail trong che do Project.
Global SearchMulti-keyword search across Part ID, Name, Description, Originator. E.g. type sheave 22.
Tim kiem nhieu tu khoa tren Part ID, Ten, Mo ta, Nguoi thiet ke.
ℹ
Tip: Double-click any row in the catalog to quickly insert that fitting into the drawing.
Meo: Double-click vao bat ky dong nao trong catalog de nhanh chong chen fitting vao ban ve.

6. Step 4: BOM Export & Ballooning

6.1 BOM Export Window

Click "Open BOM Export" to open the BOM Matrix window.
Click "Open BOM Export" de mo cua so BOM Matrix.

How to use / Cach su dung

  1. Select BOM type: Structure (Panel) or Detail (Hull).
    Chon loai BOM: Structure (Panel) hoac Detail (Hull).
  2. Click "Scan & Count from Drawing". The plugin scans all blocks in the current drawing, counts quantities, and builds the BOM matrix.
    Click "Scan & Count from Drawing". Plugin quet tat ca block trong ban ve hien tai, dem so luong, va xay dung BOM matrix.
  3. Review the matrix. Use "Auto-Assign Positions" to auto-number fittings (001, 002, ...).
    Xem lai matrix. Dung "Auto-Assign Positions" de tu dong danh so fitting (001, 002, ...).
  4. Click "Sync Pos to CAD" to write position numbers back to block attributes in the drawing.
    Click "Sync Pos to CAD" de ghi so position nguoc lai vao attribute cua block trong ban ve.
  5. Click "Export to Excel" to generate an Excel file with 2 sheets: BOM Matrix + Part Data.
    Click "Export to Excel" de tao file Excel voi 2 sheet: BOM Matrix + Part Data.

6.2 Ballooning Tools / Cong cu Balloon

ButtonFunction / Chuc nang
Place Balloon Click on fittings one by one to place balloon labels. Focus returns to drawing automatically.
Click vao tung fitting de dat nhan balloon. Focus tu dong quay ve ban ve.
Mass Auto-Balloon Select a boundary area → plugin automatically places balloons on all fittings inside that region.
Chon vung bao → plugin tu dong dat balloon cho tat ca fitting trong vung do.

7. Block Utilities / Cong cu Block

These tools modify existing blocks without breaking references in the drawing.
Cac cong cu nay chinh sua block da co ma khong lam hong tham chieu trong ban ve.
ButtonDescription / Mo ta
Rename Block Rename a block definition. All references update automatically.
Doi ten block definition. Tat ca tham chieu tu dong cap nhat.
Redefine Blocks Update block definitions from the Library DWG files. Use when fittings have been updated in the central library.
Cap nhat block definitions tu file DWG trong Thu vien. Dung khi fitting da duoc cap nhat trong thu vien trung tam.
Replace Block Replace one block with another while preserving position and attributes.
Thay the block nay bang block khac, giu nguyen vi tri va attributes.
Change Insertion Point Move the base point (insertion point) of a block definition. All instances shift accordingly.
Di chuyen diem goc (insertion point) cua block definition. Tat ca instance dich chuyen theo.
Add Objects to Block Select additional entities and add them into an existing block definition.
Chon them entity va them chung vao block definition da co.
Extract Objects from Block Pull entities out of a block into model space while keeping the block intact.
Keo entity ra khoi block vao model space, block goc van giu nguyen.
ℹ
Tip: After clicking any Block Utility button, focus automatically returns to the drawing for object selection.
Meo: Sau khi click bat ky nut Block Utility nao, focus tu dong quay ve ban ve de chon doi tuong.

9. FAQ / Cau hoi thuong gap

Installation / Cai dat

"MCG_Show" command not found / Lenh "MCG_Show" khong tim thay
EN: The plugin is not loaded. Check these steps:
1. Verify that MCGCadPlugin.bundle folder exists at %PROGRAMDATA%\Autodesk\ApplicationPlugins\
2. Make sure PackageContents.xml is inside the bundle folder
3. Make sure the .dll file name in PackageContents.xml matches the actual DLL file in Contents/
4. Restart AutoCAD completely

VN: Plugin chua duoc load. Kiem tra cac buoc:
1. Xac nhan folder MCGCadPlugin.bundle ton tai tai %PROGRAMDATA%\Autodesk\ApplicationPlugins\
2. Dam bao PackageContents.xml nam trong folder bundle
3. Dam bao ten file .dll trong PackageContents.xml trung khop voi file DLL thuc te trong Contents/
4. Khoi dong lai AutoCAD hoan toan
Plugin loads but palette is empty / Plugin load nhung palette trong
EN: This may happen if the DLL was compiled for a different AutoCAD version. Ensure you are using the DLL built for AutoCAD 2023. Contact the Automation team for the correct version.

VN: Dieu nay co the xay ra neu DLL duoc build cho phien ban AutoCAD khac. Dam bao ban dang dung DLL build cho AutoCAD 2023. Lien he team Automation de nhan phien ban dung.
Access denied when copying to ProgramData / Bi tu choi quyen khi copy vao ProgramData
EN: You need Administrator privileges. Right-click the install.bat and select "Run as administrator". If copying manually, run Windows Explorer as administrator.

VN: Ban can quyen Administrator. Click chuot phai vao install.bat va chon "Run as administrator". Neu copy thu cong, chay Windows Explorer voi quyen admin.

Step 1: IDW Import

"Inventor is not installed" error / Loi "Inventor chua duoc cai dat"
EN: The IDW import feature requires Autodesk Inventor to be installed on the same machine. If Inventor is not available, ask someone with Inventor to run Step 1 and share the output JSON + DWG files with you. You can then proceed directly with Step 2.

VN: Tinh nang import IDW yeu cau Autodesk Inventor phai duoc cai dat tren cung may. Neu khong co Inventor, nho nguoi co Inventor chay Step 1 va chia se file JSON + DWG dau ra cho ban. Sau do ban co the chuyen thang sang Step 2.
Inventor opens but files fail to process / Inventor mo nhung file khong xu ly duoc
EN: Common causes:
1. The .idw file may reference a missing .ipt/.iam model — Inventor cannot resolve views without the source model
2. The file may be read-only or locked by another user
3. Check the AutoCAD command line for detailed error messages

VN: Nguyen nhan thuong gap:
1. File .idw co the tham chieu den model .ipt/.iam bi thieu — Inventor khong the phan giai view khi thieu model nguon
2. File co the bi read-only hoac bi lock boi nguoi khac
3. Kiem tra dong lenh AutoCAD de xem chi tiet loi
Can I use Step 2 without running Step 1? / Toi co the dung Step 2 ma khong chay Step 1 khong?
EN: Yes! As long as you have the JSON and DWG file pair. The JSON file must follow the FittingMetadata format (PartNumber, Description, Views, etc.) and the DWG file must have the same filename and be in the same folder.

VN: Duoc! Mien la ban co cap file JSON va DWG. File JSON phai theo dinh dang FittingMetadata (PartNumber, Description, Views, v.v.) va file DWG phai cung ten va nam trong cung folder.

Step 2: JSON Import

"DWG file not found" warning / Canh bao "Khong tim thay file DWG"
EN: Each JSON file must have a corresponding DWG file with the same name in the same folder. Example: if you select CAS-1234567.json, there must be a CAS-1234567.dwg in the same directory.

VN: Moi file JSON phai co file DWG tuong ung voi cung ten trong cung folder. Vi du: neu ban chon CAS-1234567.json, phai co CAS-1234567.dwg trong cung thu muc.
Block already exists / Block da ton tai
EN: If a block with the same Part Number already exists in the current drawing, the import will skip it to avoid overwriting existing data. If you need to update, use "Redefine Blocks" in the Block Utilities section.

VN: Neu block voi cung Part Number da ton tai trong ban ve hien tai, import se bo qua de tranh ghi de du lieu cu. Neu can cap nhat, dung "Redefine Blocks" trong phan Block Utilities.
Which BOM Type should I choose? / Nen chon BOM Type nao?
EN:
• Panel (Structure) — Use for fittings that go on structural panels (e.g., deck panels, bulkhead panels). Layer: MCG_Fitting_Panel (blue).
• Detail (Hull) — Use for fittings attached to hull details (e.g., brackets, lugs, stiffeners). Layer: MCG_Fitting_Detail (red).

VN:
• Panel (Structure) — Dung cho fitting tren panel ket cau (vd: deck panel, bulkhead panel). Layer: MCG_Fitting_Panel (xanh).
• Detail (Hull) — Dung cho fitting dinh kem vao chi tiet vo tau (vd: bracket, lug, stiffener). Layer: MCG_Fitting_Detail (do).

Step 3: Fitting Library

Library is empty / Thu vien trong
EN: The Master Catalog file (C:\Temp_BIM_Library\MasterCatalog.json) does not exist or is empty. Run Step 1 + Step 2 first to populate the library, or use "Add from CAD" to manually add items.

VN: File Master Catalog (C:\Temp_BIM_Library\MasterCatalog.json) khong ton tai hoac rong. Chay Step 1 + Step 2 truoc de nap du lieu, hoac dung "Add from CAD" de them item thu cong.
How to create a Project Library? / Cach tao Project Library?
EN: In the Library window, click "Create Project" in the top-right corner. Choose a save location and filename. Then switch to Project Library mode to work with it.

VN: Trong cua so Library, click "Create Project" o goc tren phai. Chon vi tri luu va ten file. Sau do chuyen sang che do Project Library de lam viec.

Step 4: BOM Export

Scan returns no results / Quet khong co ket qua
EN: The scan looks for blocks with specific attributes (PART_ID, BOM_TYPE). Blocks without these attributes will be ignored. Make sure fittings were imported through Step 2 or have the correct attributes.

VN: Quet tim block co cac attribute cu the (PART_ID, BOM_TYPE). Block khong co cac attribute nay se bi bo qua. Dam bao fitting da duoc import qua Step 2 hoac co dung cac attribute.
Excel export fails / Xuat Excel that bai
EN: The BOM export uses Microsoft Excel COM automation. Excel must be installed on the machine. If Excel is open with other workbooks, close them first, or the export may interfere.

VN: Xuat BOM dung COM automation cua Microsoft Excel. Excel phai duoc cai dat tren may. Neu Excel dang mo voi workbook khac, dong chung truoc, neu khong viec xuat co the bi anh huong.

General

EN: All operations write detailed logs via Debug.WriteLine. To view them:
1. Use DebugView (Sysinternals) — free tool, no install needed

VN: Tat ca thao tac ghi log chi tiet qua Debug.WriteLine. De xem:

EN: This is rare but can happen if:
1. The DLL version doesn't match AutoCAD 2023
2. A required dependency (.NET Framework 4.8) is missing
3. The drawing file is corrupted
Please report the issue to the Automation team with the DebugView log output and the drawing file that caused the crash.

VN: Dieu nay hiem khi xay ra nhung co the xay ra neu:
1. Phien ban DLL khong khop AutoCAD 2023
2. Thieu dependency (.NET Framework 4.8)
3. File ban ve bi hong
Vui long bao cao van de cho team Automation kem log output tu DebugView va file ban ve gay loi.
How to report a bug? / Cach bao cao loi?
EN: Contact the Automation team with:
1. Screenshot of the error message
2. DebugView log output
3. Steps to reproduce the issue
4. AutoCAD version and Windows version

VN: Lien he team Automation voi:
1. Screenshot thong bao loi
2. Log output tu DebugView
3. Cac buoc de tai tao loi
4. Phien ban AutoCAD va Windows

10. JSON Format Reference / Tham chieu dinh dang JSON

For users who want to create JSON files manually (without Step 1), here is the expected format:
Cho nguoi dung muon tao file JSON thu cong (khong qua Step 1), day la dinh dang can co:
{
    "PartNumber":   "CAS-1234567",
    "Description":  "Sheave block 22T SWL",
    "Revision":     "A",
    "Mass":         "45.200",
    "Material":     "Steel S355",
    "Designer":     "NVA",
    "Title":        "Sheave Block",
    "Views": [
        {
            "Name":    "Front View",
            "CenterX": 150.0,
            "CenterY": 200.0,
            "Width":   120.5,
            "Height":  85.3
        }
    ]
}
ℹ
Important: The JSON filename must match the DWG filename. Example: CAS-1234567.json + CAS-1234567.dwg.
Quan trong: Ten file JSON phai trung ten file DWG. Vi du: CAS-1234567.json + CAS-1234567.dwg.
